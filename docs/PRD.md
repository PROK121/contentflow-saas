# ContentFlow — PRD (ядро продукта)

## 1. Видение

Единая платформа для медиадистрибьюторов и агентств в СНГ: от лида до подписанного договора, учёта лицензий, роялти и налоговых удержаний при выплатах нерезидентам.

## 2. Целевая аудитория

- Владелец бизнеса (админ) — контроль выручки, рисков, портфеля лицензий.
- Менеджеры (2+ места) — воронка, переговоры, задачи, дедлайны.
- Правообладатели — прозрачность: где продан контент, за сколько, выплаты.
- Клиенты (покупатели лицензий) — ограниченный доступ к каталогу, офферам и документам по своим сделкам.

## 3. Роли

| Роль | Код в системе | Описание |
|------|---------------|----------|
| Админ | `admin` | Полный доступ, настройки, пользователи, интеграции, отчёты. |
| Менеджер | `manager` | CRM, сделки, контракты (в рамках политики), контент (создание/редактирование черновиков), задачи. |
| Правообладатель | `rights_owner` | Только свой контент, свои лицензии/выплаты, уведомления. |
| Клиент | `client` | Просмотр разрешённых каталожных позиций, своих сделок, подписание (через интеграцию), счета при наличии прав. |

## 4. Функциональные модули

### 4.1 CRM + сделки

- **Лиды:** источник, контакт, компания, ответственный менеджер, теги.
- **Сделки:** привязка к воронке `lead → negotiation → contract → paid`.
- **Связь с контентом:** одна сделка может ссылаться на несколько `CatalogItem` (пакет) или на один; фиксируется коммерческое предложение (snapshot условий).
- **Обязательные поля сделки:** клиент, валюта предложения, ориентировочная территория, ответственный, ожидаемая дата закрытия.

### 4.2 Контент / каталог (RightsHub) — MUST

Каждый объект каталога (`CatalogItem`):

- Название, внутренний код, тип: `video` (фильмы) | `series` | `animation` (мультфильмы/аниме) | `concert_show` (концерты, шоу).
- Правообладатель (связь с организацией/контрагентом).
- **Доступные права (лицензионный пакет):**
  - Территория (справочник + произвольная детализация).
  - Срок (дата начала / окончания или длительность).
  - Формат лицензии: `exclusive` | `non_exclusive` | `sole`.
  - Платформа: `TV` | `OTT` | `YouTube` | комбинации.
- Дополнительно (юридика, см. раздел 5.3): сублицензия, языковые права, базовая территория «СНГ / Мир».

Версионирование метаданных каталога — отдельные ревизии с автором и временем.

### 4.3 Контракты

- Генерация из шаблонов (Jinja/Handlebars или аналог на бэкенде).
- Хранение PDF в объектном хранилище; хеш файла для целостности.
- **Версии:** v1 draft → v2 после правок; подписанная версия неизменяема (новые изменения — новая версия).
- Автополя: территория, срок, стоимость (из сделки/оффера), стороны, валюта, курс (если зафиксирован).
- Статусы: `draft` | `signed` | `expired` (+ опционально `terminated` для досрочного прекращения).

### 4.4 Финансы / роялти

- Учёт входящих платежей от клиентов (инвойсы, статусы оплаты).
- Выплаты правообладателям: график, удержания, нетто.
- Модели: **фикс**, **процент (royalty)** от выручки/сбора, комбинированно.
- **Задолженности:** к нам / нам должны — по контрагенту и по сделке.
- Связь строки выплаты с контрактом и каталожными позициями.

### 4.5 Dashboard

**Владелец:** выручка (период), воронка сделок, активные лицензии (по датам окончания), задолженности, топ контрагенты.

**Менеджер:** мои сделки, задачи на сегодня/неделю, дедлайны контрактов и оплат, просрочки.

### 4.6 Личный кабинет правообладателя (ключевая ценность)

- Список контента с фильтрами.
- Где продано: территория, платформа, клиент (по политике конфиденциальности — возможно агрегировано).
- За сколько: сумма сделки / доля правообладателя.
- Выплаты: история, предстоящие, удержания.
- Статусы: активная лицензия, на продлении, истекла.

### 4.7 RBAC

Матрица прав хранится в политиках (Casbin/OPA или таблица permission × role). См. `DATA_MODEL.md`.

### 4.8 Workflow / задачи

- Напоминания об окончании контрактов (за N дней).
- Дедлайны оплат (клиент → мы, мы → правообладатель).
- Renewals: флаг «требуется продление», задача менеджеру.

### 4.9 API / интеграции

- **1С / бухгалтерия:** выгрузка проводок, контрагентов, счетов (JSON/CSV или очередь событий).
- **E-signature:** webhook статуса подписи, сохранение подписанного PDF.
- **Email / WhatsApp Business API:** уведомления о статусах сделки, контракте, выплате (с согласиями и отпиской).

## 5. Нефункциональные требования (кратко)

- Аудит действий с финансовыми и юридическими сущностями.
- Хранение ПДн в РФ/регионе по требованию заказчика (on-prem / dedicated).
- Экспорт отчётов для правообладателя в PDF/Excel.

## 6. Метрики успеха

- Time-to-contract (лид → signed).
- Доля сделок с заполненным каталогом прав без ручных правок в Word.
- NPS правообладателей по прозрачности выплат.
