# Техническая архитектура ContentFlow

## 1. Обзор

- **Стиль:** модульный монолит или небольшие сервисы (Core API, Documents, Notifications) — на старте достаточно монолита с чёткими пакетами.
- **API:** REST + webhooks; в перспективе GraphQL для сложных дашбордов (опционально).
- **БД:** PostgreSQL (транзакции, JSONB для снимков коммерческих условий).
- **Очереди:** Redis / RabbitMQ / SQS — для генерации PDF, рассылок, синхронизации с 1С.
- **Файлы:** S3-совместимое хранилище (PDF контрактов, сертификаты).
- **Поиск (позже):** OpenSearch для каталога.

## 2. Сервисные границы (логические)

| Модуль | Ответственность |
|--------|-----------------|
| **Identity** | SSO-ready, JWT, refresh, приглашения, MFA для админов |
| **CRM** | Лиды, сделки, стадии, история |
| **RightsHub** | Каталог, LicenseTerms, версии метаданных |
| **Contracts** | Шаблоны, рендер PDF, версии, статусы |
| **Billing** | Инвойсы, платежи, роялти, задолженности |
| **Tax Engine** | Расчёт удержаний по правилам и профилям |
| **Workflow** | Задачи, напоминания, cron jobs |
| **Integrations** | Адаптеры 1С, e-sign, email, WhatsApp |

## 3. Безопасность

- Шифрование в покое для чувствительных полей (номера договоров можно оставить открытыми, ПДн — по политике).
- RBAC + проверка владения ресурсом (`rights_owner` видит только контент своей `organization_id`).
- Аудит: append-only таблица или стрим событий.

## 4. Интеграции

- **E-signature:** абстракция `SignatureProvider` (DocuSign / СКБ Контур / локальный провайдер); единый webhook endpoint.
- **1С:** исходящие события `DealWon`, `ContractSigned`, `PaymentReceived`, `PayoutApproved` + идемпотентные consumer’ы на стороне интеграции.
- **Уведомления:** шаблоны в БД, провайдеры Email / WhatsApp с rate limit.

## 5. Развёртывание

- Docker Compose для dev; Kubernetes для prod (по необходимости).
- Миграции: Flyway или Prisma Migrate / Liquibase.

## 6. Стек (рекомендация)

- **Backend:** Node.js (NestJS) или Go — быстрый I/O, хорошая экосистема для PDF и интеграций.
- **Frontend:** React/Next.js или Vue; отдельный портал правообладателя с упрощённой навигацией.
- **i18n:** ru-KZ, ru-RU, en — для интерфейса и шаблонов договоров.
