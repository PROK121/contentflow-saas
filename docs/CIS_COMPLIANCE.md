# СНГ: налоги, валюты, юридические атрибуты

## 1. Налог у источника (withholding)

### 1.1 Казахстан (плательщик — резидент РК, получатель — нерезидент)

- Типовые ставки удержания по роялти/лицензионным вознаграждениям зависят от **наличия и условий ДВН** (Double Tax Treaty) и классификации дохода.
- В продукте задаётся **диапазон 10–20%** как конфигурируемый справочник: для каждой пары «страна плательщик — страна получатель — тип дохода» хранятся:
  - `default_rate_without_treaty`
  - `treaty_rate_if_applicable`
  - `requires_dt_certificate`

**Поведение системы:**

1. У контрагента заполняется `TaxProfile` (резидентство, наличие сертификата резидентства/ДВН, срок действия документа).
2. При расчёте выплаты движок налогов выбирает ставку:
   - если сертификат действителен и ДВН применим → `treaty_rate`;
   - иначе → `default_rate_without_treaty`;
   - если юрист задал `withholding_rate_override` — используется он (с обязательным комментарием в аудите).
3. Результат: `withholding_tax_amount`, `amount_net`, ссылка на применённое правило и версию справочника ставок.

### 1.2 Российская Федерация

- Для ряда выплат нерезидентам применяется **налог у источника** (часто ориентир **20%** для лицензионных/роялти в общем случае — уточняется типом дохода и ДВН).
- Обязательное хранение: **сертификат резидентства** (или иной документ, подтверждающий статус для применения льготной ставки по ДВН), срок действия, файл в хранилище.

**Поведение системы:** аналогично KZ: валидация срока сертификата, предупреждение за30 дней до истечения, блокировка автоматической льготной ставки без валидного документа (переключение на консервативную ставку по умолчанию).

## 2. Мультивалюта

- Поддерживаемые коды: **USD, KZT, RUB, EUR** (расширяемо).
- Источник курсов: ЦБ РФ, НБ РК, ECB или коммерческий провайдер — конфигурируется.
- **Фиксация в контракте:** поля `fx_rate_fixed`, `fx_locked_at`, `fx_quote_currency`, `fx_base_currency` — при подписании контракта курс может «замораживаться» для расчётов по этому договору.

## 3. Юридическая модель (обязательные поля)

В метаданных лицензии и в тексте договора (автополя) должны присутствовать:

| Поле | Описание |
|------|----------|
| `territory` | СНГ, отдельные страны, «мир», с исключениями при необходимости |
| `exclusivity` | exclusive / non-exclusive / sole |
| `sublicensing` | разрешена / запрещена / ограниченно (текст) |
| `language_rights` | языки, дубляж, субтитры, оригинал |

## 4. Отчётность для аудита

- Журнал: кто изменил налоговый профиль, какая ставка применилась к каждой выплате, какая версия справочника налогов использовалась.
- Экспорт пакета документов для бухгалтерии: договор, акт, счёт, расчёт удержания.
